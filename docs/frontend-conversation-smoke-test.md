# File Agent 前端对话烟测手册

- 适用范围：已完成阶段的普通用户产品验收，以及后台低耗摘要边界验证
- 唯一主入口：浏览器 `/login`、`/getting-started`、`/chat`
- 数据库：使用当前开发测试数据库，不重建、不清空、不切换、不执行迁移
- 上位要求：用户只通过上传和自然语言使用系统，不理解 Skill、Tool、目录或数据库

## 1. 验收原则

阶段二及之前阶段的手工烟测必须通过前端页面完成。直接调用 API、执行 SQL、手工插入记录或直接修改
文件系统，不能作为普通用户产品能力的通过证据。

自动化 pytest、前端 build、后端接口测试仍然保留，但它们只用于开发回归和故障定位，不能代替以下
页面烟测。页面失败后可以只读查看浏览器 Network、服务器日志和数据库事实定位原因，但不能绕过页面
调用接口把用例补记为通过。

以下数据库行为需要区分：

- 允许：用户通过页面注册、登录、上传、发消息、确认计划产生正常业务数据。
- 禁止：为了让烟测通过而手工更新用户角色、任务状态、分类、OperationPlan 或文件记录。
- 禁止：重建数据库、清表、切换临时 SQLite、执行 Alembic upgrade/downgrade 或替换当前数据库。
- 保留：烟测数据使用明确批次标识，测试结束后不直接从数据库删除。

如果当前开发数据库 schema 与运行代码不兼容，停止烟测并报告环境问题；本手册不授权修改数据库。

## 2. 当前前端能力与验收缺口

| 能力 | 当前页面状态 | 阶段验收处理 |
|---|---|---|
| 注册、登录、进入 `/chat` | 已有 | 通过页面烟测 |
| 上传附件并发送自然语言任务 | 已有 | 通过页面烟测 |
| 重复文件三种决策 | 已有确认卡 | 通过页面烟测 |
| 逐文件整理结果 | 已有任务回执卡 | 通过页面烟测 |
| 低置信度和同名冲突提示 | 已展示待决策项 | 同名冲突选择纳入页面执行验收 |
| 重命名 OperationPlan | 已有计划与确认卡 | 通过页面烟测 |
| 有附件但不输入任务文字 | 当前输入框阻止空消息发送 | 当前阶段的明确产品约束，应验证不能提交 |
| 同名冲突自然语言执行选项 | 已完成 | 必须生成真实 OperationPlan 并从页面确认 |
| 回收站和恢复的聊天闭环 | 已完成 | 必须通过聊天生成计划并从页面确认 |
| 分类接受、拒绝和纠正页面 | 尚未完成 | 前端缺口，不得用 API 替代 |
| admin/ops 管理页面 | 尚未完成 | 前端缺口，不属于普通用户阶段二页面烟测 |

## 3. 环境使用规则

### 3.1 数据库

继续使用项目当前配置的开发测试数据库。执行人只记录以下信息，不修改配置：

```text
测试日期：
Git commit：
操作系统：
浏览器：
当前 DATABASE_URL（隐藏密码）：
测试用户：
烟测批次号：
```

批次号建议使用 `UI-SMOKE-年月日-时分`。测试文件正文中加入该批次号，避免当前数据库已有文件干扰
重复检测、同名冲突和搜索结果。

如果固定测试用户名已经存在，可以直接从页面登录；需要隔离新数据时，从 `/login` 页面注册带批次后缀
的新用户。不得通过 SQL 修改用户。

### 3.2 服务

按项目当前配置启动 API、文件 worker、生命周期 scheduler 和前端。是否启动 Neo4j 由当前开发环境决定，
不能成为阶段二上传整理的前置条件。

浏览器访问 `http://127.0.0.1:5173/login`。能够完成登录并进入 `/chat` 是本轮页面健康检查；不再把
`curl /api/health` 作为普通用户烟测步骤。

### 3.3 测试文件

所有文件必须使用虚构内容，并在正文加入本次批次号：

| 编号 | 文件 | 内容要求 | 验证目标 |
|---|---|---|---|
| UI-F01 | `通知.pdf` | 明确年份、单位、完整标题、批次号 | 泛化文件名自动整理 |
| UI-F02 | `奖学金材料.docx` | 多段落、明确业务主题和批次号 | DOCX 读取、分类、命名 |
| UI-F03 | `统计表.xlsx` | 两个以上 Sheet，含批次号、人数或金额 | 全 Sheet 读取 |
| UI-F04 | `说明.txt` | 明确标题、年份、单位和批次号 | TXT 成功路径 |
| UI-F05 | `扫描件.txt` | 内容短，缺少年份和标题 | 低置信度保留原名 |
| UI-F06 | UI-F04 的字节级副本 | 内容完全相同 | 重复上传确认 |
| UI-F07/UI-F08 | 内容不同但产生相同建议名 | 同年份和标题，不同批次内标记 | 同名冲突提示 |
| UI-F09 | 损坏 PDF | 扩展名允许但内容无效 | 单文件失败隔离 |
| UI-F10 | 加密 PDF | 需要密码 | 加密风险和停止解析 |
| UI-F11 | `.xlsm` | 宏标记 | 宏风险提示、不执行宏 |
| UI-F12 | `.doc` | 可由 LibreOffice 转换 | 旧 Word 完整读取 |
| UI-F13 | `.xls` | 两个以上 Sheet | `.xls -> 临时 .xlsx` 完整读取 |

## 4. 页面烟测用例

### UI-SMOKE-001 注册、登录与页面边界

步骤：

1. 打开 `/login`。
2. 从页面注册本次烟测用户，或登录已有开发测试用户。
3. 完成或跳过 `/getting-started`，进入 `/chat`。
4. 刷新页面，确认登录和会话可以恢复。
5. 退出后重新登录。

通过标准：

- 普通用户可以进入 `/chat`。
- 页面不出现 Skill、Tool、LangGraph、AgentRun、ToolInvocation、数据库表名、服务器路径或 Prompt。
- 不需要用户创建 workspace、填写 root_key 或选择处理流水线。

### UI-SMOKE-002 上传并通过对话自动整理

步骤：

1. 在 `/chat` 选择 UI-F01、UI-F02、UI-F03、UI-F04。
2. 在输入框输入：`请读取并自动整理这些文件。`
3. 点击发送，停留在当前会话等待任务更新。
4. 任务完成后刷新页面，再次查看本轮消息和结果。
5. 点击每个可打开的文件卡，确认页面可以访问对应文件。

通过标准：

- 每个文件都有独立结果，不能只显示“成功 4 个”。
- 结果包含整理后的名称、分类、年份、关键词、实体、风险、失败或待确认说明。
- 高置信度文件使用正文依据生成名称，扩展名保持真实格式。
- 文种不被机械追加到文件名，标题原有“通知、报告”等词不重复。
- 页面明确说明受管原件保持不变。
- 刷新后结果仍存在，不依赖前端临时状态。

### UI-SMOKE-003 附件必须同时输入任务文字

目标操作：只选择 UI-F04，不输入任何文字，确认页面不会提交；随后输入“读取并整理这个文件”再发送。

通过标准：

- 只选择附件而不输入文字时，发送按钮不得提交任务。
- 输入明确任务文字后才允许发送附件并生成最终逐文件回执。
- 无附件且无文本时同样禁止提交。

这是当前阶段的刻意约束，不是缺陷。后续只有用户明确调整产品要求后，才允许增加附件空文本提交和
系统默认任务；不得由前端自行猜测用户希望分类、总结还是仅保存文件。

### UI-SMOKE-004 旧格式和完整内容读取

步骤：

1. 上传 UI-F12 和 UI-F13。
2. 输入：`读取并整理这两个文件，分别告诉我每个文件读取了哪些内容。`
3. 完成后继续输入：`把刚才 Excel 的每个工作表分别概括一下。`
4. 再输入：`重新读取刚才两个文件。`

通过标准：

- `.doc` 的正文可以读取，重复读取能够复用结果。
- `.xls` 的所有工作表都有结果，不能只显示第一个 Sheet。
- LibreOffice 不可用时页面显示明确失败，不能伪造读取成功。
- 页面不展示转换临时路径或内部解析器名称。
- 原 `.doc`、`.xls` 的不变性由页面回执和自动化原件保护测试共同确认。

### UI-SMOKE-005 低置信度、冲突与单文件失败隔离

步骤：

1. 同一条消息上传 UI-F05、UI-F07、UI-F08、UI-F09 和一个有效 TXT。
2. 输入：`请自动整理这批文件。`
3. 等待全部文件结束，刷新页面复查结果。

通过标准：

- UI-F05 保留上传名称，并显示命名依据不足。
- UI-F07/UI-F08 不自动增加版本后缀，不覆盖已有工作副本。
- 页面展示“同时保留、保留已有、替换已有工作副本、删除已有工作副本”等允许选项。
- UI-F09 失败不影响有效 TXT 和其他文件完成。
- 直接回复“同时保留”，页面必须展示“同名文件处理计划”；确认前文件名不变，确认后才生成第二版后缀。
- 另建冲突批次分别回复“保留已有文件”“用新文件替换已有文件”“删除已有文件”，每次都必须先展示
  OperationPlan。替换或删除已有文件时，旧工作副本进入可恢复回收站，受管原件不变。

### UI-SMOKE-006 重复上传确认卡

步骤：

1. 先完成 UI-F04 的上传整理。
2. 再从页面上传字节完全相同的 UI-F06。
3. 等待“检测到相同或高度相似文件”确认卡。
4. 使用三次独立上传分别点击：
   - `取消本次上传`
   - `使用已有文件`
   - `继续上传并独立保留`
5. 每次决策后刷新页面，确认状态可以恢复。

通过标准：

- 确认卡只展示当前用户可识别的脱敏候选。
- 取消不会产生新的活动工作副本。
- 使用已有文件不会重复导入。
- 只有继续上传才进入独立归档和整理。
- 一个文件等待决策时不阻塞同批其他文件。

### UI-SMOKE-007 加密和宏风险

步骤：

1. 上传 UI-F10 和 UI-F11。
2. 输入：`读取并整理这两个文件。`
3. 等待逐文件结果。

通过标准：

- 加密文件提示需要可读取版本，不尝试破解。
- 宏文件提示宏风险，但不声称完成病毒扫描。
- 页面不出现宏执行结果、脚本输出或文件内部嵌入指令被执行的迹象。
- 一个风险文件不影响同批其他文件。

### UI-SMOKE-008 对话上下文和附件指代

在同一会话依次输入：

```text
总结刚才上传的 PDF。
把刚才表格的每个工作表分别概括一下。
重新分类第二个文件。
告诉我上一个文件的整理结果。
```

通过标准：

- “刚才、第二个、上一个”只对应后端真实解析的会话附件。
- 不能把多个歧义文件合并执行。
- 结果仍然逐文件展示。
- 普通回复不暴露内部文件 ID、路径、Skill 或 Tool。

### UI-SMOKE-008A 后台低耗摘要与主动 LLM 总结边界

前置配置：保持 `LLM_ENABLED=true`，同时设置：

```text
DOCUMENT_SUMMARY_PROVIDER=extractive
CLASSIFICATION_SUMMARY_PROVIDER=extractive
CHAT_DOCUMENT_SUMMARY_PROVIDER=llm
```

步骤：

1. 在 `/chat` 上传一份包含标题、多个章节和明确业务主题的 DOCX 或 TXT。
2. 输入：`请读取并自动整理这个文件。`
3. 等待逐文件整理回执，确认摘要、分类和关键词可以展示。
4. 只读观察模型服务调用计数或后端审计日志，确认上传、导入和分类期间没有摘要模型请求。
5. 在同一会话输入：`请详细总结刚才上传的文件。`
6. 确认页面得到基于完整正文的总结；此时允许聊天摘要 Provider 产生模型请求。
7. 把 `CHAT_DOCUMENT_SUMMARY_PROVIDER` 临时设为 `disabled` 并按正常方式重启服务后，再从页面提出同样
   的总结请求，确认系统安全降级而不是伪造模型总结。完成后恢复开发环境原配置。

通过标准：

- `LLM_ENABLED=true` 不会隐式触发后台文件双摘要模型调用。
- 后台抽取式摘要中的关键句和引用都能在原文定位，摘要缺失信息不能被解释为原文不存在。
- 只有用户明确提出总结/讲解时才允许聊天摘要 LLM 调用。
- 全部任务仍从前端消息发起；模型计数和日志只用于只读核验，不直接调用 API、不修改数据库。

### UI-SMOKE-009 重命名建议和确认

步骤：

1. 在聊天中输入：`给刚才这些文件生成标准化文件名建议，但先不要改。`
2. 查看逐文件 before/after、待确认项和风险说明。
3. 确认前刷新页面并重新打开计划，确认仍处于等待确认状态。
4. 勾选本次需要执行的文件，取消一个文件的勾选。
5. 点击页面中的确认重命名按钮。
6. 完成后通过对话输入：`显示刚才这些文件现在的名称。`

通过标准：

- 点击确认前文件没有变化。
- 只执行勾选项，取消勾选项保持不变。
- 单项失败不阻塞其他文件。
- 确认后页面显示真实逐文件执行状态，不能仅根据计划总状态猜测。
- 受管原件不变，稳定工作副本 ID 不变。

### UI-SMOKE-009A 移入回收站并恢复

步骤：

1. 在已有附件的会话中输入：`把刚才上传的文件移入回收站。`
2. 检查页面展示“移入回收站计划”，确认前仍能正常打开该工作副本。
3. 点击“确认移入回收站”。
4. 输入：`恢复刚才删除的文件。`
5. 检查页面展示“恢复文件计划”，确认前文件仍处于回收站状态。
6. 点击“确认恢复”，再通过对话读取文件。

通过标准：

- 两次物理动作都必须经过独立 OperationPlan 确认，普通消息本身不能直接移动文件。
- 确认移入回收站后页面提示文件可恢复；确认恢复后文件重新可访问。
- 恢复目标被占用时使用稳定备用位置，不覆盖其他工作副本。
- 全程不展示回收站绝对路径、工作副本存储根、Tool 或数据库 ID。
- 受管原件的路径、文件名和字节保持不变，不提供永久删除按钮。

### UI-SMOKE-010 刷新、历史消息和普通用户输出

步骤：

1. 在任务处理过程中刷新页面。
2. 任务结束后再次刷新。
3. 退出并重新登录，重新打开同一会话。
4. 滚动加载更早消息。

通过标准：

- 上传附件、重复确认状态、任务结果和 OperationPlan 可以恢复。
- 页面不依赖只存在浏览器内存中的结果。
- 普通历史消息仍只返回和展示 `task_result` 业务投影。
- 页面中不出现 Tool、Skill、AgentRun、ChangeSet、绝对路径、API key 或正文全文。

### UI-SMOKE-011 用户隔离的页面行为

步骤：

1. 从页面退出用户 A。
2. 注册或登录用户 B。
3. 在用户 B 的 `/chat` 输入用户 A 文件的批次号、整理后文件名和业务主题进行查找。
4. 检查用户 B 的历史会话和附件列表。

通过标准：

- 用户 B 不能看到、打开或下载用户 A 的文件。
- 用户 B 的查找结果不能泄漏用户 A 的文件名、分类、摘要或存在性。
- 接口级 403/404 边界由自动化权限测试保护，不作为本页面烟测的人工操作步骤。

### UI-SMOKE-012 主题检索与清空对话

步骤：

1. 在 `/chat` 依次输入“关于科研的文章有哪些”“列出与科研有关的文档”。
2. 确认回复是文件检索结果或明确的“未找到相关文件”说明，不能回复“我已收到，请继续说明你的需求”。
3. 确认当前会话已有消息后，点击左侧“清空对话”，在浏览器确认提示中继续。
4. 刷新 `/chat`，再发送一条普通消息。

通过标准：

- “文章”“列出”等自然表达进入文件检索，不要求用户理解目录或内部搜索能力。
- 清空后聊天记录不再展示，刷新后也不会恢复；新消息可以正常发送。
- 确认提示明确说明：清空对话不会删除已上传、归档或已整理的文件。
- 不显示 AgentRun、内部任务、数据库表名或本地路径。

## 5. 不允许用接口替代的未完成用例

以下能力在前端闭环完成前必须保持“未通过/未实现”，不能用 curl、Postman、Swagger 或 SQL 操作后改成
“烟测通过”：

1. 在分类卡中接受、拒绝和纠正分类。
2. 通过 admin/ops 页面查看处理状态、反馈和设置。

这些项目需要后续前端和对话编排开发；后端已有接口只能说明底层能力存在，不能说明普通用户产品闭环
已经完成。

## 6. 技术回归与页面烟测的关系

发布判断采用两条独立证据：

```text
自动化回归通过
+
前端对话烟测通过
=
对应阶段可以验收
```

自动化回归负责保护数据库审计、跨用户权限、原件 SHA-256、Chunk/Evidence 幂等和物理副作用等页面
无法直接证明的安全事实。前端烟测负责证明普通用户可以真正完成任务。任何一条失败都不能用另一条替代。

页面烟测完成后只记录：

```text
用例编号：
结果：通过 / 失败 / 前端缺口 / 环境阻塞
页面截图：
用户可见结果摘要：
失败发生在哪一步：
对应 request_id（仅故障定位时记录）：
```

不得在烟测记录中粘贴 JWT、密码、服务器绝对路径、文件正文或数据库连接密码。
